# Cursor Rules: HTML + Python 2.7 Template Generator

## CONTEXT
- Файлы независимы друг от друга (нет связей, импортов, общей логики).
- Каждый файл содержит:
  - HTML-разметку
  - Встроенный Python 2.7
  - Шаблонный синтаксис, похожий на Jinja2, но НЕ Jinja2

## PRIMARY GOAL
Агент должен:
1. Анализировать существующие файлы
2. Выявлять повторяющиеся паттерны
3. Генерировать НОВЫЕ файлы с версткой
4. Строго соблюдать выявленный стиль и синтаксис

## INPUT FOR GENERATION
При создании нового файла используются:
- правила и модель, расположенные в папке model

## CRITICAL CONSTRAINT
❗ PDF/DOCX определяет:
- структуру
- порядок блоков
- контент

❗ Существующие файлы определяют:
- синтаксис шаблонов
- способ вставки данных
- naming conventions
- формат условий и циклов

НЕЛЬЗЯ:
- придумывать новый синтаксис
- использовать чистый Jinja2
- использовать Python 3
- менять стиль шаблонов

---

## TEMPLATE SYNTAX RULES

Агент ОБЯЗАН:

1. Определить:
   - как вставляются переменные
   - как оформлены условия (if)
   - как оформлены циклы (for)
   - есть ли кастомные конструкции

2. Повторять синтаксис 1:1:
   - те же разделители
   - те же отступы
   - те же паттерны

3. Если синтаксис неочевиден:
   - НЕ угадывать
   - использовать ближайший существующий аналог

---

## HTML STRUCTURE RULES

- Строго следовать структуре из PDF/DOCX
- Не упрощать и не оптимизировать HTML
- Сохранять:
  - вложенность
  - порядок блоков
  - повторяющиеся элементы

Если макет содержит:
- таблицы → использовать таблицы
- списки → использовать списки
- блоки → повторять div-структуру

- НЕ использовать 
 - inline-стили
 - inline-css
---
## CSS RULES
- Использовать только CSS синтаксис
- НЕ использовать:
  - inline-стили
  - inline-css
  - flex
  - grid
  - border
  - padding
  - margin
  - width
  - height
  - display
  - position
  - top
  - left
  - right

## PYTHON 2.7 RULES

- Использовать только Python 2.7 синтаксис
- НЕ использовать:
  - f-strings
  - type hints
  - async/await
- Использовать:
  - классический Python (print, % formatting, etc. если встречается)

---

## PATTERN EXTRACTION

Перед генерацией агент обязан:

1. Найти:
   - повторяющиеся блоки
   - шаблонные конструкции
   - naming conventions

2. Зафиксировать:
   - как оформлены условные блоки
   - как оформлены циклы
   - как вставляются данные

3. Использовать их без изменений

---

## GENERATION PROCESS (STRICT ORDER)

1. Проанализировать текущий файл
2. Извлечь синтаксис шаблонов
3. Извлечь HTML-паттерны
4. Сопоставить с PDF/DOCX
5. Построить структуру нового файла
6. Вставить данные через существующий синтаксис
7. Проверить соответствие правилам

---

## OUTPUT RULES

- Выход = готовый файл
- Без объяснений
- Без комментариев
- Без альтернатив

---

## FORBIDDEN ACTIONS

Запрещено:
- использовать Jinja2 напрямую
- изобретать новый синтаксис
- менять стиль HTML
- оптимизировать или рефакторить
- добавлять лишние абстракции
- использовать Python 3 конструкции

---

## QUALITY CHECK

Перед завершением агент должен убедиться:

- Синтаксис совпадает с исходными файлами
- HTML соответствует макету
- Данные вставлены корректно
- Нет новых конструкций
- Код выглядит как существующий, а не сгенерированный

---

## FAILURE MODE

Если:
- данных недостаточно
- синтаксис неясен

Агент должен:
- использовать минимальные изменения
- копировать ближайший валидный паттерн
- НЕ придумывать поведение
